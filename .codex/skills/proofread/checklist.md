# Proofread checklist

What the writing-editor subagent checks on every invocation.

## Grammar & mechanics
- [ ] Typos and spelling
- [ ] Subject-verb agreement
- [ ] Tense consistency
- [ ] Punctuation (commas, semicolons, apostrophes, en/em-dashes)
- [ ] Dangling and misplaced modifiers
- [ ] Pronoun antecedents (does "it" / "they" point to something specific?)

## Clarity
- [ ] Buried subjects ("There are several reasons why…" → "Several reasons…")
- [ ] Nominalisations ("make a decision" → "decide")
- [ ] Vague qualifiers ("various", "a number of", "certain things")
- [ ] Jargon that obscures rather than condenses meaning
- [ ] Sentences over ~25 words that could be split

## Structure
- [ ] Paragraph order — does each paragraph build on the last?
- [ ] Missing transitions (does para 3 follow from para 2?)
- [ ] Duplicate ground covered in different paragraphs
- [ ] Topic sentences — does each paragraph announce what it's about?

## Tone & register
- [ ] Matches the document type (briefing note ≠ blog post)
- [ ] Consistent formality
- [ ] Hedge-stacking ("might possibly perhaps")
- [ ] AI-isms (delve, leverage, navigate the landscape, in today's fast-paced world)

## Concision
- [ ] Phrases that pad without adding meaning ("it should be noted that", "in order to", "at this point in time")
- [ ] Empty intensifiers ("really", "very", "quite", "rather")
- [ ] Redundant pairs ("first and foremost", "each and every")

## Voice preservation
- [ ] British English preserved (unless writer is American)
- [ ] Contractions kept if the writer used them
- [ ] Sentence-length pattern preserved
- [ ] Distinctive phrases kept
